Title

System and Method for Vocal Timbre Analysis and Personalized Karaoke Song Matching

------------------------------------------------------------------------

Field of the Invention

This invention relates to voice analysis and music recommendation systems, and more specifically to a method of analyzing a user’s vocal timbre and range characteristics through a humming input to generate personalized karaoke song and artist recommendations.

------------------------------------------------------------------------

Background

Existing karaoke recommendation systems rely on generic popularity rankings or basic vocal range detection (identifying the highest and lowest notes a user can sing). No existing system captures the timbral characteristics of a user’s voice — the unique color, tone, and spectral quality that distinguishes one singer from another — and uses that data to match the user to songs and artists whose vocal profiles are acoustically compatible. Furthermore, no existing system uses humming as the specific input mechanism for voice profiling, nor integrates voice biometric profiles with live venue karaoke catalog APIs to surface only songs available at a specific location in real time.

------------------------------------------------------------------------

Summary of the Invention

The invention provides a system and method for: 1. Capturing a vocal sample via humming at multiple pitch levels 2. Extracting both fundamental frequency range and spectral/timbral characteristics from the sample 3. Comparing extracted voice characteristics against a database of artist and song profiles 4. Generating ranked song recommendations based on combined range-fit and timbre-compatibility scoring 5. Optionally filtering recommendations against a live venue catalog API in real time

Claims

Independent Claims

Claim 1. A computer-implemented method for personalized karaoke song recommendation comprising: - receiving, via a microphone, one or more vocal samples from a user, wherein said samples are produced by humming at a plurality of pitch levels directed by the system; - analyzing said vocal samples to determine a fundamental frequency range comprising a lowest detectable pitch and a highest detectable pitch; - extracting one or more spectral characteristics of the vocal samples indicative of vocal timbre, including but not limited to spectral centroid, spectral rolloff, and Mel-frequency cepstral coefficients (MFCCs); - comparing the extracted frequency range and spectral characteristics against a database of song profiles, each comprising a corresponding vocal range and spectral profile; - generating a ranked list of song recommendations scored by a combination of range compatibility and timbre similarity; and - presenting said ranked list to the user as personalized karaoke recommendations.

Claim 2. The method of claim 1, wherein the vocal sample input is specifically a humming vocalization rather than a sung melodic phrase or spoken word, enabling vocal profiling without requiring the user to sing recognizable notes or lyrics.

Claim 3. A system for matching a user’s vocal characteristics to compatible musical artists comprising: - a microphone input configured to receive a hummed vocal sample; - a pitch detection module configured to identify the fundamental frequency range of the vocal sample; - a timbre analysis module configured to extract spectral characteristics from the vocal sample; - an artist profile database comprising, for each artist, a vocal range profile and a corresponding spectral/timbral profile derived from reference recordings; - a matching engine configured to score compatibility between the user’s voice profile and each artist profile using a weighted combination of range overlap and timbral similarity; and - an output interface presenting the top-matched artists and their associated songs to the user.

Claim 4. The system of claim 3, wherein the matching engine weights range compatibility and timbral similarity independently and combines them via a configurable scoring function, enabling tuning of recommendation sensitivity.

Claim 5. A method for real-time personalized karaoke song recommendation at a live venue comprising: - generating a user voice profile as recited in Claim 1; - receiving, via a network connection, a current song catalog from a karaoke venue system or catalog API; - filtering the user’s ranked recommendation list to include only songs present in the venue’s active catalog; and - presenting the filtered, personalized song list to the user in real time.

Claim 10. The method of claim 5, wherein the venue catalog is retrieved via an API integration with one or more third-party karaoke content providers, and recommendations are updated dynamically as catalog availability changes.

Brief Description of Preferred Embodiment

In a preferred embodiment, a user opens a mobile application and is guided through a three-phase humming test. In Phase 1, the user hums their lowest comfortable note for approximately three seconds. In Phase 2, the user hums at their natural resting pitch. In Phase 3, the user hums their highest comfortable note. The application captures these samples using the device microphone, performs pitch detection via autocorrelation analysis, and extracts spectral centroid values as a proxy for vocal timbre/brightness. The system then scores compatibility against a database of 24+ artist profiles and 70+ song profiles, returning a ranked list of personalized karaoke recommendations. Each recommendation displays a compatibility percentage reflecting both range fit and timbral similarity. Users may tap any recommendation to access a corresponding karaoke track.
